1. What is Podman?

Podman (Pod Manager) is a daemonless, open-source container engine for developing, managing, and running OCI containers. It provides a Docker-compatible command line interface but runs without a central daemon, offering better security and rootless operation capabilities.

Podman Architecture: ┌─────────────────────────────────────────────────────────────┐ │ USER SPACE │ │ ┌─────────────────────────────────────────────────────┐ │ │ │ podman command (CLI) │ │ │ └─────────────────────────────────────────────────────┘ │ │ │ │ │ ▼ │ │ ┌─────────────────────────────────────────────────────┐ │ │ │ conmon (container monitor) │ │ │ └─────────────────────────────────────────────────────┘ │ │ │ │ │ ▼ │ │ ┌─────────────────────────────────────────────────────┐ │ │ │ runc / crun (OCI runtime) │ │ │ └─────────────────────────────────────────────────────┘ │ │ │ │ │ ▼ │ │ ┌─────────────────────────────────────────────────────┐ │ │ │ Container Processes │ │ │ │ (direct child of podman) │ │ │ └─────────────────────────────────────────────────────┘ │ └─────────────────────────────────────────────────────────────┘

2. Install Podman

# Fedora / RHEL / CentOS sudo dnf install podman # Debian / Ubuntu sudo apt install podman # Verify installation podman --version podman info

3. Manage Podman Containers

3.1 Basic Container Operations

# Pull an image podman pull nginx:latest podman pull docker.io/library/alpine:latest # Run a container podman run -d --name mynginx -p 8080:80 nginx podman run -it --name alpine-shell alpine sh # List containers podman ps # running containers podman ps -a # all containers podman ps -a --size # with size information # Start/Stop/Restart containers podman start mynginx podman stop mynginx podman restart mynginx # Remove containers podman rm mynginx podman rm -f mynginx # force remove running container # Remove all stopped containers podman container prune

4. Container Tools: Inspect, Copy, Tag, Push, Pull

4.1 Inspect Container/Image

# Inspect container details (JSON format) podman inspect mynginx podman inspect --format '{{.NetworkSettings.IPAddress}}' mynginx # Inspect image podman inspect nginx:latest podman image inspect alpine --format '{{.Created}}' # Get container logs podman logs mynginx podman logs --tail 50 mynginx

4.2 Copy Files Between Host and Container

# Copy file from host to container podman cp /home/user/index.html mynginx:/usr/share/nginx/html/ # Copy file from container to host podman cp mynginx:/etc/nginx/nginx.conf ./nginx.conf # Copy directory recursively podman cp ./website/ mynginx:/usr/share/nginx/html/

4.3 Tag, Pull, and Push Images

# Tag an image podman tag nginx:latest myregistry.com/mynginx:v1.0 podman tag local-image:latest username/myapp:1.0 # List images podman images podman image ls # Push image to registry podman push myregistry.com/mynginx:v1.0 podman push username/myapp:1.0 docker://docker.io/username/myapp:1.0 # Pull image from registry podman pull myregistry.com/mynginx:v1.0 podman pull docker.io/ubuntu:22.04 # Login to container registry podman login docker.io podman login myregistry.com -u username -p password

5. Managing and Modifying Containers

# Execute command inside running container podman exec -it mynginx bash podman exec mynginx ls -la /etc/nginx # Update container environment variables podman run -d --name myapp -e ENV=production -e DEBUG=false nginx # Modify container resource limits podman update --cpus=2 --memory=2g mynginx # Export/Import containers podman export mynginx > mynginx.tar podman import mynginx.tar mynginx:new # Commit container to image podman commit mynginx mynginx-custom:latest

6. Rootfull Container vs Rootless Container

Rootfull vs Rootless Comparison: ┌─────────────────────────────────────────────────────────────┐ │ Rootfull Container │ Rootless Container │ │ ┌─────────────────────┐ │ ┌─────────────────────┐ │ │ │ Container runs as │ │ │ Container runs as │ │ │ │ root (UID 0) │ │ │ non-root user │ │ │ │ inside container │ │ │ mapped to host UID │ │ │ └─────────────────────┘ │ └─────────────────────┘ │ │ │ │ │ │ │ ▼ │ ▼ │ │ ┌─────────────────────┐ │ ┌─────────────────────┐ │ │ │ Runs with root │ │ │ Runs with user │ │ │ │ privileges on host │ │ │ namespaces │ │ │ │ (UID 0 mapping) │ │ │ (subordinate IDs) │ │ │ └─────────────────────┘ │ └─────────────────────┘ │ └─────────────────────────────────────────────────────────────┘

6.1 Rootfull Container

Rootfull containers run with root privileges inside the container, and the root user inside the container maps to root (UID 0) on the host. This requires sudo privileges to run.

# Run rootfull container (requires sudo or root user) sudo podman run -d --name rootfull-nginx nginx sudo podman run -it --name rootfull-shell alpine sh # Check container user (will show root) sudo podman exec rootfull-nginx whoami

6.2 Rootless Container

Rootless containers run without root privileges. The container's root user is mapped to a non-privileged user on the host using user namespaces. This is the default mode for podman when run as a regular user.

# Run rootless container (as regular user) podman run -d --name rootless-nginx nginx podman run -it --name rootless-shell alpine sh # Check user mapping podman unshare cat /proc/self/uid_map podman exec rootless-nginx whoami # Shows "root" inside container

6.3 Difference Between Rootfull and Rootless Containers

FeatureRootfull ContainerRootless Container
Privilege RequiredRequires sudo or root userCan run as regular user
UID MappingContainer root = Host root (UID 0)Container root mapped to host non-root UID
SecurityHigher risk, container escape grants host rootMore secure, container escape only grants user privileges
NetworkCan use host network, privileged ports (<1024)Cannot bind to privileged ports, uses slirp4netns
StorageCan access any host directoryLimited to user's home directory and mounted storage
PerformanceSlightly better (no namespace overhead)Minimal overhead from user namespaces
Use CaseProduction servers, system-level containersDevelopment, CI/CD, multi-tenant environments

7. How to Run Podman SQL Container (MySQL/PostgreSQL)

# Run MySQL container podman run -d \ --name mysql-db \ -e MYSQL_ROOT_PASSWORD=mysecretpassword \ -e MYSQL_DATABASE=mydb \ -e MYSQL_USER=myuser \ -e MYSQL_PASSWORD=mypassword \ -p 3306:3306 \ docker.io/mysql:8.0 # Connect to MySQL podman exec -it mysql-db mysql -u root -p # Run PostgreSQL container podman run -d \ --name postgres-db \ -e POSTGRES_USER=myuser \ -e POSTGRES_PASSWORD=mypassword \ -e POSTGRES_DB=mydb \ -p 5432:5432 \ docker.io/postgres:15 # Connect to PostgreSQL podman exec -it postgres-db psql -U myuser -d mydb

8. Running Containers Under Systemd

Podman can generate systemd unit files to manage containers as system services.

# Generate systemd unit file for existing container podman generate systemd --new --name mynginx > /etc/systemd/system/container-mynginx.service # Or for rootless containers (user systemd) mkdir -p ~/.config/systemd/user podman generate systemd --new --name mynginx > ~/.config/systemd/user/container-mynginx.service # Enable and start service sudo systemctl daemon-reload sudo systemctl enable container-mynginx.service sudo systemctl start container-mynginx.service # For rootless systemctl --user daemon-reload systemctl --user enable container-mynginx.service systemctl --user start container-mynginx.service # Check status sudo systemctl status container-mynginx.service

9. Using Pods with Podman

Pods are groups of one or more containers that share network namespace, IPC, and PID namespace. This is similar to Kubernetes pods.

Podman Pod Architecture: ┌─────────────────────────────────────────────────────────────┐ │ POD (shared namespace) │ │ ┌─────────────────────────────────────────────────────┐ │ │ │ Infra Container (pause) │ │ │ │ - Holds network namespace │ │ │ │ - Keeps pod alive │ │ │ └─────────────────────────────────────────────────────┘ │ │ │ │ │ ┌────────────────────┼────────────────────┐ │ │ ▼ ▼ ▼ │ │ ┌──────────┐ ┌──────────┐ ┌──────────┐ │ │ │ nginx │ │ php │ │ redis │ │ │ │container │ │container │ │container │ │ │ └──────────┘ └──────────┘ └──────────┘ │ │ port 80 port 9000 port 6379 │ └─────────────────────────────────────────────────────────────┘
# Create a new pod podman pod create --name webapp-pod -p 8080:80 # List pods podman pod list podman pod ps # Add containers to pod podman run -d --pod webapp-pod --name nginx-container nginx podman run -d --pod webapp-pod --name php-container php:apache podman run -d --pod webapp-pod --name redis-container redis # Inspect pod podman pod inspect webapp-pod # Execute command in pod (affects all containers) podman pod top webapp-pod # Stop and remove pod (removes all containers) podman pod stop webapp-pod podman pod rm webapp-pod # Create pod with specific settings podman pod create \ --name db-pod \ --share net,ipc,uts \ -p 5432:5432 \ -p 3306:3306

10. Build Images Using Podman and Buildah

Buildah is a tool for building OCI container images without requiring a container daemon. It works alongside Podman.

10.1 Building with Podman (using Containerfile/Dockerfile)

# Create a Containerfile cat > Containerfile << 'EOF' FROM docker.io/alpine:latest LABEL maintainer="dev@example.com" RUN apk add --no-cache nginx EXPOSE 80 CMD ["nginx", "-g", "daemon off;"] EOF # Build image podman build -t my-custom-nginx:latest . podman build -t myapp:v1.0 -f Containerfile . # Build with build arguments podman build --build-arg VERSION=1.0 -t myapp:latest .

10.2 Building with Buildah

# Install buildah sudo dnf install buildah # Create new container (working container) buildah from alpine # Mount the container buildah mount alpine-working-container # Run commands inside container buildah run alpine-working-container apk add --no-cache nginx # Copy files buildah copy alpine-working-container ./index.html /var/www/html/ # Set configuration buildah config --port 80 alpine-working-container buildah config --cmd "nginx -g 'daemon off;'" alpine-working-container # Commit to image buildah commit alpine-working-container my-custom-alpine:latest # List images buildah images

11. Complete Example: Pull, Push, and Run Container

This example demonstrates a complete workflow: pulling an image, running a container, modifying it, tagging, and pushing to a registry.

Scenario: Create a custom Nginx container with a custom index page, then push it to Docker Hub.
# Step 1: Pull base image podman pull docker.io/library/nginx:alpine # Step 2: Create custom index page cat > index.html << 'EOF' Custom Nginx

Hello from Podman Container!

This is a custom built container.

EOF # Step 3: Run container and copy custom page podman run -d --name custom-nginx -p 8080:80 nginx:alpine podman cp index.html custom-nginx:/usr/share/nginx/html/ # Step 4: Commit container to new image podman commit custom-nginx my-nginx:custom # Step 5: Tag image for registry podman tag my-nginx:custom docker.io/yourusername/my-nginx:latest podman tag my-nginx:custom docker.io/yourusername/my-nginx:v1.0 # Step 6: Login to Docker Hub podman login docker.io -u yourusername # Step 7: Push image to registry podman push docker.io/yourusername/my-nginx:latest podman push docker.io/yourusername/my-nginx:v1.0 # Step 8: Stop and remove local container podman stop custom-nginx podman rm custom-nginx # Step 9: Pull and run from registry on another machine podman pull docker.io/yourusername/my-nginx:latest podman run -d --name deployed-nginx -p 8080:80 yourusername/my-nginx:latest # Step 10: Verify curl http://localhost:8080

11.1 Example with Buildah (Image Building)

# Create working container container=$(buildah from docker.io/alpine:latest) # Install packages buildah run $container apk add --no-cache nginx curl # Create custom configuration cat > nginx.conf << 'EOF' server { listen 80; server_name localhost; location / { root /usr/share/nginx/html; index index.html; } } EOF # Copy files buildah copy $container nginx.conf /etc/nginx/conf.d/default.conf # Create custom index page echo "

Built with Buildah!

" > index.html buildah copy $container index.html /usr/share/nginx/html/ # Configure container buildah config --port 80 $container buildah config --cmd "nginx -g 'daemon off;'" $container # Commit image buildah commit $container yourusername/buildah-nginx:latest # Push to registry podman push yourusername/buildah-nginx:latest docker://docker.io/yourusername/buildah-nginx:latest # Clean up working container buildah rm $container

11.2 Example with Pod (Multi-Container Application)

# Create a pod for web application podman pod create --name webapp-pod -p 8080:80 -p 8081:8080 # Run web server container podman run -d --pod webapp-pod --name webserver nginx:alpine # Run application container (custom app) podman run -d --pod webapp-pod --name app-container alpine sh -c "while true; do echo 'App running' >> /var/log/app.log; sleep 5; done" # Run database container podman run -d --pod webapp-pod --name database --env MYSQL_ROOT_PASSWORD=secret mysql:8.0 # Check pod status podman pod ps podman pod inspect webapp-pod # Access services curl http://localhost:8080

11.3 Cleanup Commands

# Remove containers podman rm -f custom-nginx deployed-nginx webserver app-container database # Remove pod podman pod rm -f webapp-pod # Remove images podman rmi my-nginx:custom podman rmi yourusername/my-nginx:latest podman rmi yourusername/buildah-nginx:latest # Remove all unused containers, images, and networks podman system prune -a
